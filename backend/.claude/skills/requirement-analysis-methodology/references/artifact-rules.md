# Artifact Rules

## Generate a version snapshot when

- the project first becomes understandable to another teammate
- a major ambiguity is resolved
- MVP direction changes materially
- an artifact is generated successfully

## Generate a document draft when

- requirement summary is stable enough to hand off
- scope and boundary are mostly clear
- open questions can be listed explicitly without blocking the whole draft

## Generate a page plan when

- the target product shape is clear enough to describe screens or modules
- the user needs alignment on information architecture

## Generate an interaction draft when

- a key flow is clear enough to walk through step by step
- misunderstandings are likely if only prose is shown

## Do not generate artifacts just because the user has chatted for a while

Artifacts should reduce ambiguity, not create more of it.

客户需求转译台

客户需求转译台是一个面向 AI 项目前期的需求分析智能体工作台。它接住聊天、文档、图片等原始输入，在多轮澄清中持续沉淀项目画像、范围边界、待确认项、MVP 方案和原型建议，把“客户说得很乱的需求”转成“产品、设计、研发、售前都能继续推进的执行稿”。

5 阶段工作流 三栏工作台 需求分析智能体 可导出 PDF / PRD
最新主界面采用更真实的制造业售后项目数据：12 个服务站、28,416 条工单、4 套系统、知识命中率 61%、试点 2 站、预计 ROI 6.8 个月。
problem

要解决的核心问题

输入碎片化

客户原话、会议纪要、文档、图片、流程说明分散，前期上下文难稳定保留。

需求不成型

客户表达的常常是愿望、抱怨或方案倾向，不是能直接交给交付团队的执行需求。

分析质量不稳定

顾问、售前、产品经理的方法论不一致，导致 AI 适配判断、范围边界和优先级反复变化。

从分析到交付断层

需求分析后很难直接过渡到页面结构、原型建议和设计交付稿。

positioning

产品定义

它是什么

一个“需求分析 + 方案收敛 + 设计交付”的阶段化工作台。

它不是什么

不是通用聊天机器人，不是单纯文档总结工具，也不是直接写代码的开发平台。

核心原则

分析优先、生成其次；AI 起草、人确认排版；每条重要结论尽量能回溯资料来源。

核心输出

项目卡片、项目画像、待确认项、真实需求、MVP 方案、原型建议、设计交付稿。

users and scenarios

目标用户与使用场景

直接用户

AI 转型咨询顾问、售前顾问、产品经理、解决方案架构师、企业内部数字化负责人。

典型场景

售前首次接单、咨询需求澄清、方案定义、设计交付、项目预研立项。

核心价值

降低需求澄清成本、统一分析结构、减少重复沟通，让结果直接进入设计与研发评估。

stage model

五阶段工作流

1. 需求接入

接住客户原话和初始资料，建立项目卡片与缺失资料清单。

输入

聊天、会议纪要、介绍文档、基础资料

系统动作

抽取要点、识别问题方向、建立项目卡片

输出

项目卡片、问题定义、缺失资料清单

2. 业务理解

还原角色、流程、术语、系统和约束，把业务讲明白。

输入

流程资料、角色资料、系统清单、访谈补充

系统动作

流程建模、角色抽取、系统边界识别

输出

项目画像、流程图、约束与痛点

3. 需求收敛

把事实、假设、目标和边界拆开，只围绕关键缺口追问。

输入

项目画像、冲突点、用户补充

系统动作

列待确认项、识别冲突、收敛真实需求

输出

范围边界、真实需求、优先级

4. 方案定义

判断 AI 适配度，形成 MVP 方案、能力包和风险边界。

输入

已收敛需求、数据条件、系统条件

系统动作

AI fit 评估、能力拆分、MVP 起草

输出

MVP 草案、风险边界、验收指标

5. 设计交付

把方案转成页面结构、交互流和低保真原型建议。

输入

MVP 能力、字段、关键流程

系统动作

信息架构、页面地图、交互流生成

输出

页面清单、原型建议、交付稿

ux model

界面结构

顶部状态栏

项目名、当前阶段、资料数、待确认项、AI 适配度、版本状态。

左栏：资料与证据

沉淀项目概况、资料列表、流程文件、术语、字段、页面资产和证据片段。

中栏：聊天与澄清

只问关键问题；每轮对话后尽量生成“当前理解”或“待确认项”。

右栏：项目收敛结果

沉淀已确认事实、范围边界、MVP 草案、指标卡片和原型建议。

mvp scope

MVP 范围

必须有

聊天 / 文档 / 图片输入，解析与 OCR，项目画像，待确认项，真实需求，MVP 方案，原型建议，三栏工作台。

先不做

高保真视觉自动生成、自动代码生成、深行业专家系统、自动报价 / 合同。

成功标准

需求收敛周期缩短 30%+；需求分析结果可直接进入设计或研发评估；每个阶段都有明确产物。

technical solution

技术方案

前端工作台

Next.js + TypeScript，承载三栏工作台、阶段切换、资料上传、结构化结果编辑与导出。

接入与解析层

聊天、文档、图片接入；文档抽取、OCR、图片理解、文本切块与元数据清洗。

理解层

项目画像、角色、流程、目标、系统边界、痛点、约束、风险的结构化建模。

判断层

缺口识别、冲突检测、追问生成、AI 适配判断、范围边界识别。

产出层

真实需求、待确认项、MVP 方案、原型建议、导出文档与 PDF。

存储层

PostgreSQL + pgvector 存结构化上下文与检索索引；对象存储保存原始文件；所有关键结论保留 source refs。

实现建议：不要用一个大 prompt 直接跑到底。推荐拆成 source parser、profile builder、gap detector、requirement condenser、ai fit evaluator、solution composer、design handoff generator 等模块化链路。
ai collaboration

需求分析智能体职责与人机分工

AI 负责

信息抽取、冲突识别、缺口判断、需求收敛、MVP 起草、原型建议。

AI 不拍板

最终需求确认、最终优先级、正式风险承诺、正式对外交付版排版。

人负责

确认、修改、取舍、排版与最终发布；必要时打回上阶段重新理解。

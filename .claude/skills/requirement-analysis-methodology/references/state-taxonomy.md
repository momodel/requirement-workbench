# State Taxonomy

## current_understanding

Use for the current best integrated view. It can contain informed interpretation, but it should still be evidence-aware.

## pending_items

Use when a question remains open and its answer would change:

- scope
- source-of-truth
- data mapping
- exception handling
- acceptance criteria

## confirmed_items

Use only when supported by explicit user confirmation or strong grounded evidence.

## conflict_items

Use when two or more statements cannot both be true, or when different teams use the same term with different meanings.

Typical examples:

- field definition mismatch
- accounting rule mismatch
- lifecycle mismatch
- historical docs contradicting current practice

## mvp_items

Use for what should survive into the first runnable solution after risk and effort compression.

An item belongs here only if:

- it solves a real bottleneck
- it is safe enough for first release
- it does not depend on unresolved high-risk assumptions

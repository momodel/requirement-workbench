# 客户需求转译台 一期演示脚本

## 1. 演示目标

这份脚本用于本地演示当前一期实现，不再依赖旧 demo 的分阶段页面。

建议讲法围绕一条主线：

1. 这是一个 `Project-first` 的客户需求转译台
2. 默认案例是 `业财逐笔对账`
3. 资料先进入左侧 Sources
4. 中间 Chat 负责把问题逐轮聊清楚
5. 右侧 Project State 持续沉淀理解、待确认项、MVP 和交付物

## 2. 演示前准备

### 2.1 启动后端

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### 2.2 启动前端

```bash
cd frontend
npm install
npm run dev
```

### 2.3 默认地址

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8000`

## 3. 标准演示路径

### 3.1 首页

- 打开首页，说明现在顶层对象已经是 `Project`
- 指出默认 seed project：`业财逐笔对账`
- 如果需要，也可以顺手新建一个测试项目

### 3.2 进入工作台

- 点击 `进入工作台`
- 说明左中右三栏各自职责：
  - 左边是资料
  - 中间是对话
  - 右边是持续沉淀和交付物

### 3.3 导入资料

建议至少演示两种：

1. 导入一段补充文本，例如：
   - `退款场景下，业务系统用 refund_flag 标记，财务系统通过红字科目冲销。`
2. 导入一个链接或文件

讲法重点：

- 资料会先入库
- 后端会生成标准化摘要
- 后续 NotebookLM 工作流会基于这些摘要给出 grounding

### 3.4 发起一轮聊天

建议输入：

```text
我们先关注逐笔差异识别，不自动改账。
```

观察点：

- 中间聊天区会出现 `message_chunk`
- 同时还能看到 `citations`
- 后端会发回状态 patch
- 右侧刷新后能看到新的当前理解、待确认项、已确认项、MVP 和版本快照

讲法重点：

- 这不是只回一段话
- 后端同时在维护结构化项目状态

### 3.5 生成交付物

依次点击右上角动作：

1. `文档稿`
2. `页面方案`
3. `交互稿`

观察点：

- 文档稿以结构化内容展示
- 页面方案和交互稿会以大预览层打开 HTML
- 生成结果已经落盘，不只是页面临时拼出来的字符串

## 4. 建议口播

- “左侧不是阶段材料，而是一个持续增长的项目资料库。”
- “中间这轮聊天同时驱动了引用、状态 patch 和版本快照。”
- “右侧不是静态看板，而是项目当前共识的结构化总集。”
- “交付物不是聊天记录导出，而是已经转换成文档稿和可预览的 HTML 原型。”

## 5. 异常与 fallback 说明

如果现场没有起后端：

- 首页和工作台仍可走 seed fallback
- 但聊天 patch、artifact 内容和资料接入会退回本地 fallback 模式

如果后端已起但外部 provider 不可用：

- 当前一期默认会走本地 provider
- 仍可完整演示资料入库、聊天、状态沉淀、版本快照和交付物生成

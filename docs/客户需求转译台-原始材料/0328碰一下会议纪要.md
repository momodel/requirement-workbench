# 0328 需求碰一下

时间：3 月 28 日下午  
参会：张总、李敏、陈帆、赵倩

## 先聊出来的大意

这个项目主要是想解决“客户需求前期太散、太乱、不成型”的问题。

现在很多项目不是没有机会，而是前面几轮聊不清楚：

- 客户说的是想法，不是需求
- 给的材料也不成体系
- 内部每个人抓到的信息还不一样
- 到了产品或方案阶段，又得重新整理一遍

## 这次会里提到的一些想法

### 1. 希望能收什么

- 客户原话
- 聊天记录
- 会议纪要
- 产品介绍
- 流程类材料
- 截图
- 可能还有表格

音频这次也提到了，但没有定第一版一定要不要做。

### 2. 希望系统能帮忙做什么

- 先理解项目背景
- 把角色、流程、问题点拎出来
- 自动发现还缺哪些信息
- 给出下一步该追问什么
- 把已确认和未确认的内容分开
- 最后沉淀一个比较像样的初步结果

### 3. 输出可能是什么

这块没有完全定。

会上有人提到可以有：

- 一版需求结论
- 待确认问题列表
- MVP 建议
- 页面结构草图
- 一份可以继续讨论的文档

但到底第一阶段做到哪，还要再收一收。

### 4. 界面感觉

倾向不是一个单聊天界面。

有人提到最好是：

- 一边看资料
- 一边聊天
- 一边看整理后的结论

但还没到要出详细页面的时候。

## 当前没定的

- 是不是要做成多项目管理
- 一个项目里资料怎么组织
- 输出物是不是要版本化
- 页面草图是不是第一版必须有
- 有没有必要做权限
- 后面是否要接到正式交付流程里

## 会后动作

- 先把现有零散想法和资料收一收
- 先出一个比较轻的方向，不要直接做成大而全
- 可以先拿一个案例演示，但不要把产品讲窄

# 客户需求转译台

这个名字是我们内部先这么叫，后面如果要对外再看要不要改。

我们现在遇到的情况大概是这样：

- 最近来聊 AI 的客户很多，但大部分一开始都说得比较散
- 售前、咨询、产品每次都要重复做一轮需求梳理
- 资料经常是会议纪要、PPT、微信群聊天、Word、截图混着来
- 同一个项目聊几轮以后，前面讲过的背景和边界又容易丢

所以我们想先做一个偏前期分析用的东西，先不讲做成多大平台，核心是把客户那些比较乱的表达先接住，再往下慢慢理。

现在我们自己理解，这个东西大概不是一个普通聊天框，应该更像一个工作台，但具体长什么样还没想透。

我们目前最关心的几个点：

1. 客户原话和资料能不能先放进来
2. 系统能不能帮忙把业务背景、目标、问题、边界先分开
3. 能不能边聊边沉淀，不要聊完又回到一堆记录里找
4. 最后最好能形成一个初步方案，至少能给内部继续讨论

先说下我们自己脑子里模模糊糊的使用场景：

- 售前接到一个 AI 相关机会，不知道该怎么往下接
- 咨询顾问拿到一堆材料，想先快速看懂业务
- 产品经理想把前面的零散内容整理成后面能继续推进的东西

暂时没有完全想清楚的地方：

- 是做成一个独立系统，还是先做成一个 demo
- 是偏内部使用，还是未来也可能给客户一起看
- 输出到底是 PRD、方案、页面草图，还是先只做需求结论
- AI 在里面负责到什么程度，哪些还得人工自己判断

我们现在的诉求先别拔太高，第一步更想看一个方向是不是对。
